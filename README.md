# 버튼 잡기 게임 (C# 코딩)
## 개요
- C# 프로그래밍 학습

---

- 핵심기능:
-버튼 위치 제어: MouseEnter 이벤트 발생 시 버튼이 창 영역 내부에서 무작위 좌표로 이동.

-실시간 난이도 조절: 클릭 성공 시 버튼의 크기를 일정 비율(10%)로 축소하여 난이도 상.

-점수 관리 시스템: 초기 점수 200점을 기준으로 성공 시 +100점, 실패 시 -10점을 부여하며 실시간 UI 동기화.

-예외 처리 및 게임 종료: 누적 실패 횟수가 20회에 도달할 경우 재시작/종료 창 띄우기.

-상태 초기화(Reset): 모든 게임 데이터 및 컨트롤 속성(위치, 크기, 폰트)을 초기 상태로 복구하는 기능.

---

- 화면구성:
-메인 폼 (Form1): 게임이 실행되는 기본 윈도우 창이며, 버튼이 화면 밖으로 나가지 않도록 ClientSize를 통해 이동 범위를 제한합니다.

-게임 버튼 (MyButton): 사용자가 클릭해야 할 핵심 컨트롤로, MouseEnter 시 위치가 바뀌고 Click 시 크기와 폰트가 동적으로 축소됩니다.

-타이틀 바 (this.Text): 폼의 제목 표시줄에 실시간 누적 점수, 현재 버튼의 X/Y 좌표, 누적 실패 횟수를 텍스트로 실시간 업데이트합니다.

-알림창 (MessageBox): 버튼 클릭 성공 시의 피드백을 전달하고, 게임 오버 시 최종 점수 출력 및 Yes/No 버튼을 통해 재시작 여부를 결정합니다.

## 구현 시 어려웠던 점
-1. 버튼이 폼 밖으로 나가지는 문제.
해결: $maxX = ClientWidth - ButtonWidth$ 공식을 적용하여 버튼이 항상 화면 안에서만 랜덤하게 생성되도록 설계했습니다.

-2. 버튼 크기가 10%씩 작아질 때 글자(폰트) 크기는 자동으로 변하지 않아 버튼 밖으로 글자가 튀어나가는 문제.
해결: MyButton.Font를 새로 생성하여 MyButton.Font.Size * 0.9f 비율로 버튼과 함께 축소되도록 구현했습니다.


-3. 한 번도 잡지 못하고 20번을 놓쳤을 때 최종 점수가 0점이 아닌 10점으로 표시되는 오류가 발생.
해결: 점수 감점 코드(score -= 10)의 위치를 if 조건문 위로 배치하여, 20번째 카운트가 되는 즉시 점수가 먼저 깎인 후 메시지 박스가 뜨도록 순서를 교정하여 정확히 0점이 표시되게 만들었습니다.

-4. 재시작 시 게임을 다시 시작할 때 버튼의 크기와 폰트만 돌아오고 위치가 고정되어 있는 것.
해결: RestartGame() 함수 내에 랜덤 위치 생성 로직을 포함시키고, 반드시 버튼 크기를 먼저 키운 후 좌표를 계산하여 버튼이 화면 밖으로 삐져나가는 예외를 방지했습니다.
